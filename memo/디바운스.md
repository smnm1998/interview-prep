# 디바운스(debounce)

> 학습일 2026-07-21 · 구글링하며 풀어서 개념 정리 후 cold 재테스트 예정

## 디바운스란?

> "마지막 호출 이후 조용해질 때까지 기다렸다가, 딱 한 번 실행"

디바운스는 **연달아 쏟아지는 함수 호출을, 마지막 호출을 기준으로 일정 시간 조용해질 때까지 기다렸다가 딱 한 번만 실행**하게 만드는 기법이다. 핵심 단어는 "마지막"과 "한 번"이다. 호출이 계속 들어오는 동안에는 실행을 자꾸 뒤로 미루다가, 더 이상 호출이 없으면 그제서야 진짜로 실행한다.

머릿속에 엘리베이터 문을 떠올리면 쉽다. 사람이 계속 타는 동안은 문이 닫히지 않고, 아무도 안 들어오고 몇 초가 지나야 비로소 닫힌다. 디바운스도 똑같이 "입력이 멈추고 잠잠해지는 순간"을 기다린다.

실무에서 가장 흔한 자리는 검색 자동완성이다. 사용자가 `리액트`를 칠 때 글자마다 API를 때리면 `ㄹ`, `리`, `리ㅇ`, `리애`… 매번 요청이 나간다. 디바운스를 걸면 타이핑이 멈춘 뒤 300ms쯤 지나 마지막 단어로 한 번만 요청한다. 그 외에도 `resize`/`scroll`처럼 폭풍처럼 발생하는 이벤트나, 버튼 연타 방지에 자주 쓴다.

## 어떻게 동작하나

내가 짠 버전은 이른바 **trailing edge**(뒷쪽 실행) 방식이다. 호출이 들어올 때마다 이전에 잡아둔 예약을 취소하고 새 예약을 건다. 그래서 실행 시점이 계속 뒤로 밀리다가, 마지막 호출 이후로 조용한 시간이 `wait`만큼 흐르면 그때 실행된다.

```
호출:   a     b     c
              ↑ b가 오면서 a 예약 취소, 다시 타이머
                    ↑ c가 오면서 b 예약 취소, 다시 타이머
                    └────── 500ms 조용 ──────▶ ✅ c 실행
```

즉 a와 b는 영원히 실행되지 않고, 마지막 c만 살아남는다. "조용해진 뒤 한 번"이라는 정의가 이 취소-재예약 반복에서 나온다.

## 코드로 뜯어보기

내가 작성한 코드는 이렇다.

```js
const debounce = (func, timeout) => {
    let timer; // 예약 id를 기억할 자리
    return (...args) => {
        // 실제로 여러 번 호출될 "디바운스된 함수"
        clearTimeout(timer); // 직전 예약 취소
        timer = setTimeout(() => {
            // 새 예약
            func.apply(this, args); // wait 뒤 원래 함수 실행
        }, timeout);
    };
};
```

겉보기엔 짧지만 이 안에 자바스크립트 핵심 개념 네댓 개가 겹쳐 있다. 하나씩 풀어본다.

## 밑에 깔린 개념

### 클로저 — timer가 살아있는 이유

`debounce`는 호출되는 순간 `timer`라는 지역 변수를 만들고, 그 아래에서 함수를 하나 만들어 리턴한다. 보통 함수가 끝나면 그 안의 지역 변수는 사라져야 정상인데, 리턴된 함수가 `timer`를 계속 쓰고 있기 때문에 `timer`는 사라지지 않고 살아남는다. 이렇게 **함수가 자신이 태어난 스코프의 변수를 계속 기억하는 것**을 클로저라고 한다.

```js
function counter() {
    let n = 0;
    return () => ++n; // n을 계속 기억한다
}
const c = counter();
c(); // 1
c(); // 2   ← n이 안 사라진다
```

디바운스에서 `timer`가 바로 이 `n`이다. 디바운스된 함수를 몇 번을 호출하든 전부 **같은 하나의 `timer`**를 바라보기 때문에, 이전 예약을 정확히 찾아 지우고 새로 걸 수 있다. 클로저가 없으면 매 호출마다 `timer`가 초기화돼서 취소 자체가 불가능하다.

### 고차 함수 — 함수를 리턴하는 함수

`debounce`는 원래 함수 `func`을 받아서 **새로운 함수를 돌려준다**. 이렇게 함수를 인자로 받거나 함수를 리턴하는 함수를 고차 함수라 부른다. 그래서 `debounce(log, 500)`의 결과를 변수에 담아두고, 그 변수를 이벤트 핸들러처럼 여러 번 부르는 사용 패턴이 자연스럽게 나온다.

### setTimeout과 clearTimeout — 예약과 취소

`setTimeout(fn, ms)`은 "`ms` 뒤에 `fn`을 실행해줘"라고 예약하고, 그 예약을 가리키는 **타이머 id**를 반환한다. 이 id를 `clearTimeout(id)`에 넘기면 아직 실행 안 된 예약을 취소할 수 있다. 디바운스의 리듬은 결국 **매 호출마다 이전 id를 취소하고 새 id를 발급받는 것**의 반복이다.

한 가지 더 알아두면 좋은 건 타이밍이다. `setTimeout` 콜백은 지금 실행 중인 동기 코드가 전부 끝난 다음에야 실행된다. 그래서 아래 코드에서 `d("a"); d("b"); d("c")`는 순식간에 다 돌아가고, 실제 `func` 실행은 그 후 500ms 뒤에 일어난다.

```js
const log = (msg) => console.log("실행:", msg);
const d = debounce(log, 500);
d("a");
d("b");
d("c");
// 동기 코드는 여기서 끝. 500ms 뒤 → "실행: c" 한 번만 찍힘
```

### 나머지 인자와 apply — 인자 그대로 넘기기

디바운스된 함수는 몇 개의 인자로 불릴지 모른다. 그래서 `(...args)`로 들어온 인자를 **전부 배열로** 받아두고(rest 파라미터), 나중에 실행할 때 `func.apply(thisArg, args)`로 그 배열을 원래 함수에 **펼쳐서** 넘긴다. `apply`는 첫 인자로 `this`를, 둘째 인자로 인자 배열을 받는다. 인자를 하나씩 넘기고 싶으면 `func.call(thisArg, ...args)`를 써도 결과는 같다.

## this 함정: 화살표 vs 일반 함수

내 코드에서 아직 미묘하게 어긋난 부분이 바로 이 `apply(this, args)`의 `this`다. 리턴하는 함수를 **화살표 함수**로 썼는데, 화살표 함수는 자기만의 `this`가 없다. 대신 **정의된 위치의 바깥 `this`를 그대로 물려받는다.** 반면 일반 함수 `function(){}`의 `this`는 **어떻게 불렸는지(호출 방식)**에 따라 그때그때 정해진다.

```js
const obj = {
    name: "Kim",
    regular() {
        console.log(this.name);
    }, // this = obj  → "Kim"
    arrow: () => console.log(this.name), // this = 바깥 → undefined
};
obj.regular(); // "Kim"
obj.arrow(); // undefined
```

즉 디바운스된 함수를 `obj.greet()`처럼 메서드로 불러도, 리턴 함수가 화살표라면 그 안의 `this`는 `obj`가 아니라 정의 시점의 바깥 컨텍스트가 된다. `apply(this, ...)`로 `this`를 챙기는 것처럼 보이지만 실제로는 엉뚱한 `this`를 넘기고 있는 셈이다. 이 문제는 **리턴 함수를 화살표 대신 일반 함수 표현식으로 바꾸면** 호출자의 `this`가 제대로 잡히면서 해결된다. (완성 코드는 직접 고쳐볼 것 — 아래 참고.)

## 관련 개념

디바운스와 자주 헷갈리는 것이 **스로틀(throttle)**이다. 디바운스가 "조용해진 뒤 한 번"이라면, 스로틀은 "일정 간격마다 주기적으로"다. 스크롤 위치를 100ms마다 갱신한다든지, 게임에서 발사를 0.2초에 한 번으로 제한하는 식이다. 엘리베이터 문이 디바운스라면, 스로틀은 몇 분마다 한 대씩 오는 지하철 배차에 가깝다.

실행 시점도 옵션이 갈린다. 내가 짠 건 마지막에 실행하는 **trailing**이지만, 첫 호출에서 즉시 한 번 실행하고 이후 조용해질 때까지 무시하는 **leading** 방식도 있다. 라이브러리(lodash 등)는 보통 둘을 옵션으로 함께 준다. 또 실전에서는 예약된 실행을 취소하는 `.cancel()`이나 지금 당장 실행하는 `.flush()` 같은 메서드를 덧붙이는데, 특히 컴포넌트가 사라질 때 남은 타이머를 정리해 메모리 누수를 막는 용도로 `.cancel()`이 중요하다.

## 한 줄 요약

디바운스 = **클로저로 타이머 id를 기억하다가, 호출마다 이전 예약을 취소하고 새로 걸어서, 마지막 호출 이후 조용해지면 딱 한 번 실행**하는 것.
